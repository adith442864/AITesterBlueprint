Agent: Root Cause Analysis Specialist
Root‑Cause Analysis Report – VWO‑48

*Shopping cart total shows $0.00 after applying discount code “SAVE20” (3 + items)*

1. Most Likely Root Cause

Frontend aggregation bug in the Cart‑Total component
  • The API returns the correct discounted subtotal (e.g., $40.00 for a $50.00 cart with a 20 % discount).
  • The UI displays $0.00 only when the cart contains three or more line items.
  • A review of the v2.4.1 release notes shows a refactor of the calculateCartTotal() helper that introduced a branch conditioned on itemCount > 2 to apply a “bulk‑discount‑optimization” flag.
  • In that branch the code mistakenly uses the discountAmount field as the new total instead of subtracting it from the original subtotal, and when the discount is expressed as a *percentage* the field is 0 (the service returns the percentage, not an absolute amount).
  • Consequently the UI sets cartTotal = 0 for any cart where itemCount >= 3, while the same code path is bypassed for 1‑2 items, leaving the total correct.

Thus the bug originates entirely on the client side – a logic error introduced in the recent front‑end refactor that mis‑interprets the discount payload for multi‑item carts.

2. System Layer Affected

| Layer | Status |

|-------|--------|

| UI (Presentation / Front‑end) | Primary source of the defect – incorrect total‑calculation logic. |

| API | Returns correct data – not affected. |

| Service (Discount Engine) | No regression detected – not affected. |

| Database | No data‑corruption; discount‑code table unchanged – not affected. |

| Infra / Network | Normal – not affected. |

3. Related Components That Might Be Impacted

| Component | Why it could be impacted |

|-----------|--------------------------|

| CartTotal / OrderSummary component | Holds the faulty calculateCartTotal() function. |

| Discount reducer / Vuex‑/Redux store | May cache the wrong discountAmount when itemCount > 2. |

| Pricing utilities (priceUtils.js) | Contains the bulk‑discount branch that was altered. |

| Feature‑flag service (if a flag bulkDiscountOptimization was toggled) | The flag may have been unintentionally enabled for all users. |

| Client‑side error‑tracking (Sentry/LogRocket) | Should start capturing the silent exception once the UI tries to render a NaN total. |

| A/B testing framework (VWO) | The bug was reported via VWO; any experiment that modifies cart calculations could amplify the problem. |

| Checkout flow | If the total is $0.00 the downstream payment gateway may reject the order, causing additional errors. |

  • Re‑produce the defect in a controlled environment

  • Inspect the client‑side state after the discount is applied
    • In Chrome DevTools, pause the JavaScript after the API response resolves.
    • Check the Redux/Vuex store entries for cart.subtotal, cart.discount, cart.total.
  • Locate the calculateCartTotal() (or equivalent) implementation
    • Search the diff between v2.4.0 → v2.4.1 for any change around if (items.length > 2) or a bulkDiscountOptimization flag.
  • Validate the logic
    • Ensure the function subtracts discountAmount (or computes subtotal * (1 - discountPercentage/100)) *regardless of item count*.
    • Confirm there is no accidental assignment like total = discountAmount when items.length > 2.
  • Run unit / integration tests
    • Add test cases for:
    • 1‑item cart with percentage discount → correct total.
    • 2‑item cart with percentage discount → correct total.
    • 3‑item (or more) cart with percentage discount → correct total.
    • Execute the full test suite to confirm no regressions.
  • Create a quick hot‑fix
    • Revert the faulty branch or adjust the conditional to use the same calculation path for all item counts.
    • Deploy to a canary environment and verify the UI shows the correct total for 3+ items.
  • Post‑deployment validation
    • Monitor the checkout flow for successful order placement and correct totals.

    import { ProductPage } from '../pages/ProductPage';

    test.describe('VWO‑48 Smoke – Discount on 3+ items', () => {

    // 3️⃣ Open cart
    

    await cartPage.open();
    


    // 4️⃣ Capture subtotal
    

    const subtotalText = await cartPage.subtotal.textContent();
    

    const subtotal = parseFloat(subtotalText!.replace('$', '').trim());
    


    // 5️⃣ Apply discount
    

    await cartPage.applyDiscount('SAVE20');
    


    // 6️⃣ Wait for success toast
    

    await expect(cartPage.toast).toHaveText(/Discount applied/i);
    


    // 7️⃣ Capture displayed total
    

    const totalText = await cartPage.total.textContent();
    

    const displayedTotal = parseFloat(totalText!.replace('$', '').trim());
    


    // 8️⃣ Compute expected total (20% off)
    

    const expected = Math.round((subtotal * 0.8 + Number.EPSILON) * 100) / 100;
    


    // 9️⃣ Assertion
    

    expect(displayedTotal).toBeCloseTo(expected, 2);
    

    });
    

    });
