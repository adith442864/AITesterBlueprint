What QA/SDET should understand first

  • LLM apps are probabilistic: expected outputs are ranges, not exact strings.
  • Quality is not enough: you also test safety, grounding, cost, latency, and reliability.
  • For RAG systems, always separate indexing tests from retrieval/generation tests.

Transformer Paper (Attention Is All You Need) in QA Terms

The Transformer replaced recurrence with attention, enabling better parallelism and stronger long-range modeling. For QA teams, this directly affects long-context behavior, latency scaling, and output consistency under varying prompt length.

Input Tokens -> Embedding + Positional Encoding -> Encoder Stack (Self-Attention + FFN) -> Decoder Stack (Masked Self-Attention + Cross-Attention) -> Linear + Softmax -> Next Token Prediction
Test idea: Evaluate output quality at 1k, 4k, 8k, and 16k token inputs.
Test idea: Reorder paragraph blocks and measure semantic drift.
Risk: Longer prompts can increase latency and reduce grounding precision if context assembly is weak.

QA/SDET Playbook (Quick Checklist)

  1. Functional: Golden prompts + rubric scoring.
  2. RAG: Precision@k, recall@k, citation correctness, hallucination rate.
  3. Security: Prompt injection, jailbreak, tool abuse tests.
  4. Performance: p50/p95 latency, throughput, token-cost tracking.
  5. Reliability: Retry/backoff, timeouts, degraded-mode behavior.
  6. Observability: Request-level traces and alerting thresholds.
