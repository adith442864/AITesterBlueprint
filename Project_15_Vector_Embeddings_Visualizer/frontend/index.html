How to explain this live

  • Chunking is how we keep each vector focused on a manageable slice of meaning.
  • An embedding is a numeric signature. Similar meanings usually sit closer together.
  • The heatmap compares chunks using cosine similarity.
  • The 2D plot is only a teaching projection. The real vector still has dozens or thousands of dimensions.
For workshops, start with Deterministic Demo or Ollama / Nomic. Switch to OpenAI or Mistral when you want a real hosted provider.

Teaching notes

1. Split

Chunking creates small retrieval units from longer documents.

2. Embed

Each chunk becomes a vector so machines can compare meaning numerically.

3. Compare

Cosine similarity shows which chunks are close in vector space.

4. Retrieve

Later, a user query can be embedded and matched against these chunk vectors.
